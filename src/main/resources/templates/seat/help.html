📖 좌석 관리 테스트 도움말

🎫 테스트 페이지로 이동 📋 API 문서 (Swagger)

🎯 테스트 페이지 주요 기능

  • 좌석 시각화: 150개 좌석을 A, B, C 구역으로 구분하여 표시
  • 실시간 상태 확인: 좌석의 현재 상태를 색상으로 구분 표시
  • 좌석 선점/해제: 클릭 한 번으로 좌석 선점 및 해제 가능
  • 실시간 폴링: 2초 간격으로 자동 상태 업데이트
  • TTL 모니터링: 선점한 좌석의 남은 시간 실시간 표시
  • 다중 사용자 테스트: 사용자 ID 변경으로 동시성 테스트 가능

🚀 사용 방법

1. 기본 설정

  • 콘서트 ID: 테스트할 콘서트의 고유 번호 (기본값: 1)
  • 사용자 ID: 테스트할 사용자의 고유 번호 (기본값: 1)

2. 좌석 선점 테스트

  • 🟢 녹색 좌석 (예약 가능): 클릭하면 선점됩니다
  • 🔵 파란색 좌석 (내가 선점중): 클릭하면 해제됩니다
  • 🟡 노란색 좌석 (다른 사용자 선점중): 선택할 수 없습니다
  • 🔴 빨간색 좌석 (예매 완료): 선택할 수 없습니다

3. 실시간 폴링

  • 폴링 시작: "실시간 폴링 시작" 버튼 클릭
  • 자동 업데이트: 2초마다 모든 좌석 상태가 자동으로 갱신됩니다
  • 폴링 중지: "실시간 폴링 중지" 버튼 클릭

4. 동시성 테스트

  • 다중 탭 열기: 같은 페이지를 여러 탭에서 열기
  • 다른 사용자 ID: 각 탭에서 다른 사용자 ID 설정
  • 동시 선점 시도: 같은 좌석을 동시에 클릭하여 분산 락 테스트

🔧 사용된 API 엔드포인트

참고: 모든 API는 RESTful 방식으로 구현되어 있으며, 분산 락(Redisson)을 통해 동시성 문제를 해결합니다.

⚠️ 주의사항

  • TTL 만료: 선점한 좌석은 5분 후 자동으로 해제됩니다
  • 분산 락: 같은 좌석에 대한 동시 요청은 순차적으로 처리됩니다
  • 네트워크 오류: 서버 연결 실패 시 적절한 오류 메시지가 표시됩니다
  • 브라우저 호환성: 모던 브라우저에서 최적화되어 있습니다

🧪 테스트 시나리오 예시

시나리오 1: 기본 선점/해제 테스트

  1. 콘서트 ID: 1, 사용자 ID: 1로 설정
  2. A구역의 임의 좌석(예: A-10) 클릭
  3. 좌석이 파란색으로 변경되는지 확인
  4. 같은 좌석을 다시 클릭하여 해제
  5. 좌석이 녹색으로 돌아가는지 확인

시나리오 2: 동시성 테스트

  1. 새 탭을 열어 같은 페이지 접속
  2. 첫 번째 탭: 사용자 ID 1, 두 번째 탭: 사용자 ID 2
  3. 동시에 같은 좌석 클릭
  4. 한 사용자만 선점 성공하는지 확인
  5. 다른 사용자에게는 적절한 오류 메시지 표시 확인

시나리오 3: TTL 만료 테스트

  1. 임의 좌석 선점
  2. 실시간 폴링 시작
  3. 5분 대기 (또는 서버에서 TTL 시간 단축 설정)
  4. 만료 시 자동으로 좌석이 해제되는지 확인

🔍 트러블슈팅

문제: 좌석 상태가 업데이트되지 않음

  • 해결방법: "좌석 상태 새로고침" 버튼 클릭
  • 원인: 네트워크 지연 또는 서버 응답 지연

문제: 좌석 선점이 실패함

  • 해결방법: 다른 사용자가 선점했는지 확인
  • 원인: 분산 락으로 인한 동시성 제어

문제: 페이지가 느림

  • 해결방법: 실시간 폴링 중지 후 수동 새로고침 사용
  • 원인: 2초 간격의 지속적인 API 호출
